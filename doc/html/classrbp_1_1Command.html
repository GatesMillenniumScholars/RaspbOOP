Detailed Description

Encapsulates data into a command.

This class is responsible for the encoding and decoding of data into and out of the Raspboop Transmission Protocol (RTP). The RTP is formatted as follows:

{ START OF PACKET, BODY LENGTH, COMPONENT ID, COMMAND ID, PARAMETERS... }

Name Definition Memory Usage
START OF PACKET An arbitrary value that starts every packet 1 byte
BODY LENGTH The size of the remaining portion of the data 1 byte
COMPONENT ID An ID representing a physical component 1 byte
COMMAND ID The ID of a Component's command 1 byte
PARAMETERS An array of floats 8*x bytes
  • The RTP also specifies another arbitrary value that can commence a packet of data: a CONNECTION REQUEST

Command takes care of formatting the data into the RTP specification.
